SummaTRON-API

Access API to the identification system created by SummaTRON, type 2FA and based on the TRON Blockchain to improve the identification of users on the web pages.

There are more than 1.06 billion web pages and more than 28 million registered web domains. The SummaTRON API wants to help simplify and increase the security of pages with user identification.

The identification API works with a 2FA system, double factor identification, based on the fact that a user who has a TRON account in a wallet and sends an ITX Token to another account, can only be him. If you have lost control of your account, you have lost your identity and its content (equivalent to losing a digital certificate along with the password).

On the option AVA you can test the operation. Sending 1 IdTronix you can see your account information without having to use the private key. This way if you use Windows it would not be necessary to type anything. It works like browsing Tronscan, but without typing the private key. Does not replace wallet functionality: shipping or payment.

Based on this premise, any company that has or wants to implement a user identification system can use user and password: SMS, Google Authenticator, Microsoft Authenticator, etc., SummaTRON will be a similar and even more simple method.

For example, a company that wants to use it only has to integrate the API and register, which means:

  • 1) Create or have a TRON account, you must buy 1 SummaTRON and the IdTronix that you think is necessary for your number of users, its price is 0.001 TRX.

  • 2) The registration process consists in sending the SummaTRON to the project account: TFQwLDzUvEc99ktd3TvUc9g3uATGmX2fS7, and very important in the description indicate the name of the web, for example: EmpresaXX.

  • 3) Initiate your users to install a wallet (TronWalletMe has the PAY option that streamlines the process by scanning all the data at once).

  • 4) In the installation of the wallet, users will create a TRX account.

  • 5) On the web users must register that TRX account, which will be used to identify themselves.

  • 6) The company will send some ITX to that account so that the user initially does not have to do anything, and can already use them to identify themselves.

  • 7) The user when he wants to enter the website (which will have already implemented the SummaTROn API) will show QRs, scan and send an ITX that will go to the company's own account, so the company will only need to buy more ITX if increase your customers.


If the TRON System succeeds and the use of cryptocurrencies and wallet is generalized, the same users can buy the ITX, otherwise the company would move all that it buys among its clients, it will need as many as daily identifications. More activity, more ITX than you need, but the cost compared to SMS is 100,000 less, since ITX are reusable.

In the BODY, where you want to show the QR for control the access

	
	<div id="PanelIRS" style="display:block;"></div> 
	//With this css style  control then QR visible or not.

At the end of the HTML insert this 

	  
	<script>
	   $("#PanelIRS").load("/api/PanelIRS.html", function() {
	   sCuenta = "XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX";
	   Verificar(sCuenta,"EN", "FUNCTION to EXECUTE");
	   });
	</script>
With this script, the QR is displayed and the TRON account of the company is indicated.
In FUNCTION to EXECUTE from, you must indicate the name of the function that will be executed when the identification ends, it is returned as parameter the TRON account, or an empty string, if after 60 seconds the identification has no occurred.

Why TRON?

Currently a large amount of user information and internet traffic is concentrated in a few corporations such as Google, Facebook, Amazon and Snapchat. TRON tries to mitigate this control by giving ownership of the information returned to its creators.

TRON is open and decentralized platform and distributed storage technology will allow digital content creators to eliminate intermediaries such as Apple Store and Google Play. In this way content producers obtain their profits directly from consumers, without intermediaries.


Electronic books, music, photos, articles, etc. they can be easily marketed without intermediaries to make the payment.

Banks, Visa, Amazon, Google, Facebook, etc. they take much of the amount of operations, basically because until now there was no alternative.

The transactions carried out in TRON are traceable, through Tronscan.org, the 2 participants in the purchase / sale can see the result of the operation.

This functionality replaces the need to contact your bank or Visa, when a transaction is "lost". But it does not replace the possible scammers, TRON improves technology, not the morality of people.

How to use TRON?
            

TRON was born with the aim of assembling an architecture based on blockchain and that manages to execute 10,000 transactions per second.
Its currency is called Tronix (TRX), and 100,000 million have been issued.

The TRXs are stored in an electronic wallet, wallet in English. Wallet for mobile, allow Payment, Sending or reception of TRX with other accounts in trones.

With this capacity it can be used as a means of payment in all types of transactions, but also as a form of identification without the need for registration, since only the user has access to his TRX account.


A System capable of executing operations in less than 1 second has a multitude of utilities.
